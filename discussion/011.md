## Bug: Stream decoder accepts invalid tag syntax `!key id` instead of `!key(id)`

The stream decoder incorrectly parses `!key id [...]` (space-separated tag argument) instead of rejecting it.

### How it parses

```tony
{x: !key id [1]}
```

Gets parsed as:
- Key: `x`  
- String `"id"` with tag `!key`
- Array `[1]` as a **separate sibling value**

### Impact

When sent to logd via session protocol:
1. Patch is accepted (should be rejected)
2. Data corruption - patch structure is malformed
3. Existing data may be overwritten unexpectedly

### Correct syntax

Use parentheses: `!key(id) [...]`

### Fix needed

Stream decoder should either:
1. Reject `!key id` as invalid syntax (preferred)
2. Or parse it consistently as `!key(id)` if both syntaxes should be supported
